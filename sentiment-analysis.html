Entity Level 'X' Sentiment Analysis

Problem Statement

Businesses today face several challenges in monitoring their brand reputation, struggle to pinpoint specific areas driving negative or positive sentiment within their brand's ecosystem.

Data Used

The dataset for this project was obtained from Kaggle and contained 4 features. Key features included:

  • entity: Specific brand or entity mentioned in the tweet
  • sentiment: Human labeled sentiment Possible Values: 'Positive', 'Negative', 'Neutral', 'Irrelevant'
  • tweet: The raw text content of the X post

Initial exploration revealed a fairly balanced dataset.

Methodology

The project followed a standard data science lifecycle:

  1. Data Cleaning & Preprocessing:
    • Handled missing values, Lowercasing, punctuation removal.
    • Noise removal and Stopwords from NLTK removal
    • Tokenization and Lemmatization using NLTK
  2. Exploratory Data Analysis (EDA):
    • Visualized Sentiment Distribution.
    • Visualized Entity Distribution.
    • Visualized Tweet length Distribution
  3. Model Selection & Training:
    • Logistic Regression with TF-IDF
    • Random Forest classifier with TF-IDF
  4. Model Evaluation:
    • Generated Confusion matrix to access performance visually
    • Performed Classification report to get the precision, recall, F1-scores

Results & Insights

The Logistic Regression emerged as the best-performing model, achieving a strong 80% accuracy.

This project successfully developed and evaluated an entity-level X sentiment analysis framework. This capability empowers businesses to move beyond superficial social listening to truly understand public opinion at a micro-level, enabling more targeted and effective strategic responses.

Tools & Technologies

  • Languages: Python
  • Libraries: Pandas, NumPy, Scikit-learn, Matplotlib, Seaborn, NLTK
  • Development Environment: Jupyter Notebooks
  • Version Control: Git, GitHub
